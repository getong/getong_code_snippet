* injectable and get_it

`get_it` is a service locator for Dart and Flutter.
`injectable` is a code generator built on top of `get_it`.

Use `get_it` when you want a central place to register and resolve dependencies.
Use `injectable` when you do not want to manually write all `get_it.register...` calls.

Common idea:

1. `get_it` stores objects and factories.
2. `injectable` reads annotations and generates registration code.
3. Your app calls one bootstrap method during startup.

** when to use them

Good fit:

- app has repositories, services, API clients, local storage, analytics, auth
- app wants constructor injection instead of creating dependencies inside widgets
- app has multiple implementations for dev, test, prod

Not a good fit:

- very small app with only a few classes
- app already uses another dependency injection pattern and adding this would duplicate concepts

** package roles

`get_it` gives you:

- singleton registration
- lazy singleton registration
- factory registration
- instance lookup with `getIt<T>()`

`injectable` gives you:

- annotations like `@injectable`, `@singleton`, `@lazySingleton`
- generated registration code
- environment-based registration
- async dependency setup with `@preResolve`

** basic get_it usage

Create one global locator:

#+begin_src dart
import 'package:get_it/get_it.dart';

final getIt = GetIt.instance;
#+end_src

Register dependencies (v9.2.1):

#+begin_src dart
void setupDependencies() {
  getIt.registerLazySingleton<ApiClient>(() => ApiClient());
  getIt.registerLazySingleton<AuthRepository>(
    () => AuthRepositoryImpl(getIt<ApiClient>()),
  );
  getIt.registerFactory<LoginCubit>(
    () => LoginCubit(getIt<AuthRepository>()),
  );
}
#+end_src

Resolve dependencies:

#+begin_src dart
final cubit = getIt<LoginCubit>();
#+end_src

Registration types:

- `registerSingleton`: create immediately, reuse the same instance
- `registerLazySingleton`: create on first use, then reuse (most common)
- `registerFactory`: create a new instance every time
- `registerSingleton(..., instanceName: 'name')` (v9.2.1): register multiple instances

New in get_it 9.2.1:

- `getIt.asNewInstance<T>()`: register a factory that creates new instances
- Better null safety and type inference
- ~5-15% faster lookup performance
- Enhanced error messages for debugging

** why add injectable

Without `injectable`, you write and maintain registration code by hand.
With `injectable`, you annotate classes and let code generation build the wiring.

That means:

- less boilerplate
- easier refactoring
- clearer constructor-based dependency graph

** common annotations

`@Injectable()`

- registers a class as a factory (creates new instance each time)

`@Singleton()`

- one eager instance for the app lifetime, created at startup

`@LazySingleton()`

- one shared instance, created only when first needed

`@Singleton(as: SomeType)` or `@LazySingleton(as: SomeType)`

- register implementation as an interface or abstract type

`@factoryMethod`

- tells `injectable` which constructor or static creator to use

`@PostConstruct()`

- run setup logic after object creation

`@preResolve`

- await an async dependency before registration completes (requires async `configureDependencies`)

`@factoryParam`

- pass runtime parameters into a factory instance

** module example (async external services)

Use a module when initializing external services like Firebase. The module approach is cleaner than annotating third-party classes:

#+begin_src dart
import 'package:firebase_core/firebase_core.dart';
import 'package:injectable/injectable.dart';
import 'package:shared_preferences/shared_preferences.dart';

@module
abstract class ExternalServicesModule {
  @preResolve
  Future<FirebaseApp> get firebaseApp async {
    return await Firebase.initializeApp();
  }

  @preResolve
  Future<SharedPreferences> get prefs async {
    return SharedPreferences.getInstance();
  }
}
#+end_src

When using `@preResolve`, startup is async. Update `main()`:

#+begin_src dart
import 'package:flutter/material.dart';
import 'di/injection.dart';

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await configureDependencies(); // Must await
  runApp(const MyApp());
}
#+end_src

The generated code automatically initializes Firebase and SharedPreferences before your app starts.

** practical rules (v9.2.1 & v2.7.1+4)

- always use capitalized annotations with parentheses: `@Singleton()`, `@LazySingleton()`, `@Injectable()`
- prefer `@LazySingleton()` for services to avoid eager startup cost
- use `@Injectable()` for ephemeral objects (cubits, notifiers)
- use `@Singleton()` only for app-level services that must be ready immediately
- use `instanceName: 'name'` to register multiple instances of the same type (v2.7.1+)
- use `@module` for external services that need async initialization
- use `@PostConstruct()` for setup logic instead of custom constructors
- inject dependencies through constructors for better testability
- depend on abstractions (interfaces/abstract classes), not concrete types
- keep widget code free from object creation—inject via constructor
- run build runner watch during dev: `fvm dart run build_runner watch`
- wrap `main()` with `async` when using `@preResolve` dependencies
- call `WidgetsFlutterBinding.ensureInitialized()` before `await configureDependencies()`
- use `getIt.reset()` in tests to clean up between test cases

** get_it vs provider style access

`get_it` is not a widget state management solution.
It only resolves dependencies.

You often combine it with:

- `flutter_bloc`
- `provider`
- `riverpod`
- `mobx`
- plain `ValueNotifier` or `ChangeNotifier`

Typical pattern:

1. register repository or service in `get_it`
2. inject it into cubit, notifier, or controller
3. expose that state object to the widget tree

** common problems (v9.2.1 & v2.7.1+4)

Generated file missing or outdated:

- run `fvm dart run build_runner build --delete-conflicting-outputs`
- if issues persist, run `fvm dart run build_runner clean` first then rebuild
- ensure annotations use parentheses: `@LazySingleton()` not `@LazySingleton`
- check that your class has a public constructor (private constructors are ignored)

Type not found in `getIt<T>()`:

- verify the class has the correct annotation
- check the generated `injection.config.dart` includes the type
- run build runner watch to regenerate: `fvm dart run build_runner watch`
- if using environment-based registration, ensure you called `.init(environment: 'env-name')`
- for multiple instances, use correct `instanceName`: `getIt<Type>(instanceName: 'name')`
- clear pubspec lock and regenerate if version mismatch occurs

Async crash at startup:

- you used `@preResolve` but did not `await configureDependencies()`
- ensure `main()` is async: `Future<void> main() async`
- call `WidgetsFlutterBinding.ensureInitialized()` before `await configureDependencies()`
- check that all `@preResolve` dependencies actually complete

Circular dependency (v2.7.1+ is stricter):

- class A depends on B and B depends on A
- refactor shared logic into a third service
- depend on abstractions to break the cycle
- use lazy singletons or factories to defer initialization
- use `getIt` directly inside a method instead of constructor injection for one dependency

Wrong injection generated:

- verify class has `@Injectable()` or similar annotation with parentheses
- check that constants/getters in modules use `@preResolve` or return async
- abstract classes cannot be annotated—only their concrete implementations
- run `fvm dart run build_runner clean && fvm dart run build_runner build --delete-conflicting-outputs`
- update injectable_generator version to match injectable version

InstanceName not working (v2.7.1+):

- ensure you're using `@LazySingleton(instanceName: 'name')` not just `instanceName`
- resolve with exact name: `getIt<Type>(instanceName: 'name')`
- verify the name string is identical in both annotation and lookup
- check generated code to see how names are registered

** short example flow

1. run `fvm flutter pub add get_it injectable && fvm flutter pub add --dev build_runner injectable_generator`
2. create `lib/di/injection.dart` with `@InjectableInit()` setup
3. annotate services and repositories with `@LazySingleton()`
4. annotate cubits/notifiers with `@Injectable()`
5. for multiple instances, use `@LazySingleton(instanceName: 'name')`
6. for async init, use `@module` with `@preResolve` getters
7. run `fvm dart run build_runner watch` to generate code
8. wrap `main()` with `async` if using `@preResolve`
9. call `configureDependencies()` in `main()` before `runApp()`
10. resolve objects with `getIt<T>()` or inject them into higher-level objects

** summary

`get_it v9.2.1` is a production-grade service locator with excellent performance. It handles dependency lookup, registration, and resolution with ~5-15% faster performance than v7.6.0.

`injectable v2.7.1+4` auto-generates the registration code from annotations, handling the boilerplate automatically.

Together they provide:

- clean constructor-based dependency injection
- minimal boilerplate
- scalable setup for medium-to-large Flutter apps
- testable code (dependencies can be mocked)
- environment-based and instance-name-based configuration
- better error messages and debugging
- support for multiple instances of the same type
- async initialization chains for external services

For a small app, plain `get_it` is often enough.
For a growing app, `injectable + get_it` significantly reduces maintenance burden and enables compile-time safety.

** testing with injectable (v9.2.1 & v2.7.1+4)

With injected dependencies, testing becomes straightforward. Reset and re-register dependencies for each test:

#+begin_src dart
import 'package:flutter_test/flutter_test.dart';
import 'package:get_it/get_it.dart';
import 'package:mockito/mockito.dart';

class MockAuthRepository extends Mock implements AuthRepository {}

void main() {
  group('LoginCubit', () {
    late GetIt testGetIt;
    late MockAuthRepository mockAuthRepository;

    setUp(() {
      testGetIt = GetIt.instance;
      testGetIt.reset(); // Clear any previous registrations

      mockAuthRepository = MockAuthRepository();
      testGetIt.registerSingleton<AuthRepository>(mockAuthRepository);
    });

    tearDown(() {
      testGetIt.reset();
    });

    test('signIn calls repository', () async {
      final cubit = LoginCubit(testGetIt<AuthRepository>());

      when(mockAuthRepository.signIn('test@example.com', 'password'))
          .thenAnswer((_) async {});

      await cubit.submit('test@example.com', 'password');

      verify(mockAuthRepository.signIn('test@example.com', 'password')).called(1);
    });
  });
}
#+end_src

#+begin_src dart

import 'package:flutter/material.dart';
import 'package:get_it/get_it.dart';
import 'package:injectable/injectable.dart';

import 'injection.config.dart';

const String devEnvironment = 'dev';
const String prodEnvironment = 'prod';

final GetIt getIt = GetIt.instance;

@InjectableInit(
  initializerName: r'$initGetIt',
  preferRelativeImports: true,
  asExtension: false,
)
Future<void> configureDependencies(String environment) async {
  // Async now supported end-to-end
  await $initGetIt(getIt, environment: environment);
